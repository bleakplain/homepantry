## 核心内容

### What（做什么）

**核心功能**:
1. **基于库存的推荐**: 根据当前库存推荐菜谱
2. **个性化推荐**: 根据用户偏好和历史记录推荐
3. **协同过滤推荐**: 基于相似用户的推荐
4. **快速推荐**: 基于时间限制和难度等级的快速推荐

### Why（为什么）

**对用户的价值**:
- 根据现有食材快速找到菜谱
- 个性化推荐，符合用户口味
- 减少决策时间

**对产品的价值**:
- 提高用户粘性
- 差异化竞争特性
- 提高菜谱使用率

## 数据模型

### 核心组件

智能推荐功能不需要独立的数据库实体，它主要是一个业务逻辑功能，基于现有数据（Recipe, PantryItem 等）进行推荐。

**核心组件**:
1. **RecipeRecommender**: 推荐引擎（基于库存、个性化、协同过滤）
2. **RecipeRecommendation**: 推荐结果数据类
3. **RecommendationViewModel**: 推荐页面 ViewModel
